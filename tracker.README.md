# tracker 统计标记文件内容格式说明

## 直接举例
```
[
  {
    "web": ["#ffffff", 0],    # web 分类统计的颜色和分类权重，int 值越大表示越优先，初始化时不检查数据类型
    "game": ["#0000ff", 3],   # game 分类统计的颜色和分类权重
    "tool": ["#ff00ff", 1],   # tool 分类统计的颜色和分类权重
    "other": ["#656565", -1], # other 分类统计的颜色和权重，必须包含 other 分类  
  },
  {
    "chrome.exe": {               # 必须，进程名称，区分大小写且应当唯一
      "name": "Chrome Browser",   # 必须，显示在统计结果里的名称，区分大小写，不同进程使用同一名称时，将合并计算时间
      "catalog": "web"            # 可选，在分类统计中的分类，不区分大小写，将使用 .title() 处理；若不存在则视为 other
    },
    "deadcells.exe": {
      "name": "Dead Cells",
      "catalog": "game",
      "reminder_time": 0,         # 可选，达到时间限制时弹窗提醒，0为不提醒，当不存在时，使用默认值；目前暂时不使用；
      "priority": 0               # 可选，必须为 int ，初始化时不检查数据类型，易发生错误；当存在时，以此处定义的分类权重有效
    },
    "deadcells_gl.exe": {
      "name": "Dead Cells",
      "catalog": "game"，
      "reminder_time": 0,         # 可选，达到时间限制时弹窗提醒，0为不提醒，当不存在时，使用默认值；目前暂时不使用
      "priority": 0
    }
]
```

## 注意
+ 权重最小值不应小于 -9999 ，否则程序会抛出错误，在无控制台接收输出时无法发现程序已停止运行.
